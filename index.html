Algorithm Complexity Analysis

Most frequently asked in UGC NET — understand notations, master theorem, and recurrence relations.

Big-O Notation — O(f(n))

Upper Bound (Worst Case)

f(n) = O(g(n)) means there exist constants c > 0 and n₀ such that:

f(n) ≤ c·g(n)   for all n ≥ n₀

Used most commonly in practice. Gives the worst-case guarantee.

Omega Notation — Ω(f(n))

Lower Bound (Best Case)

f(n) = Ω(g(n)) means there exist constants c > 0 and n₀ such that:

f(n) ≥ c·g(n)   for all n ≥ n₀

Describes the best-case behaviour of the algorithm.

Theta Notation — Θ(f(n))

Tight Bound (Average Case)

f(n) = Θ(g(n)) means the function is both O(g(n)) AND Ω(g(n)):

c₁·g(n) ≤ f(n) ≤ c₂·g(n)

The most precise description when best = worst case.

Complexity Order (slowest to fastest growth) — Memorise this!

From FASTEST to SLOWEST (left = most efficient): O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

Case 3: f(n) = Ω(n^(k+ε)) → T(n) = Θ(f(n))

Arrays

Focus on memory address calculation formulas — guaranteed 1–2 questions in every paper.

1D Array Address Formula

Address of A[i] : Base + (i – LB) × Size
LB = Lower Bound (default 0), Size = element size in bytes

Example: A[5..15], base=1000, int(4 bytes). Address of A[10] = 1000 + (10-5)×4 = 1020

2D Array Address Formulas

Row-Major Order (C language)

Address of A[i][j] : Base + [(i – R₁)×NC + (j – C₁)] × Size
NC = Number of Columns, R₁/C₁ = lower bounds

Row is fixed, columns vary continuously. Elements A[0][0], A[0][1], A[0][2] … A[1][0] … are stored in sequence.

Column-Major Order (FORTRAN)

Address of A[i][j] : Base + [(j – C₁)×NR + (i – R₁)] × Size
NR = Number of Rows, R₁/C₁ = lower bounds

Column is fixed, rows vary continuously. Elements A[0][0], A[1][0], A[2][0] … A[0][1] … are stored in sequence.

🔥 Solved Example (asked in UGC NET)

Array A[1..4][1..5], base=100, each element 2 bytes, Row-Major. Find address of A[3][4].

= 100 + [(3-1)×5 + (4-1)] × 2  =  100 + [10+3]×2  =  100 + 26  =  126

NC = 5 (columns 1 to 5). R₁=1, C₁=1. Always subtract lower bounds from i and j.

Key Points
  • Row-major: used in C, C++, Java (row of a 2D array is contiguous in memory).
  • Column-major: used in FORTRAN, MATLAB, R.
  • Total elements in A[m..n] = n – m + 1.
  • Static arrays: fixed size at compile time. Dynamic arrays (ArrayList): resize by doubling → amortized O(1) insert.

Linked Lists

Understand types, advantages over arrays, and operation complexities.

Singly Linked List

  • Each node: data | next →
  • Last node points to NULL
  • Traverse only in one direction
  • Tail → next = NULL

Doubly Linked List

  • Each node: ← prev | data | next →
  • Can traverse both directions
  • Head → prev = NULL
  • Tail → next = NULL

Circular Linked List

  • Singly circular: tail → next = head
  • Doubly circular: tail → next = head, head → prev = tail
  • No NULL pointer
  • Used in round-robin scheduling

Key Points
  • Advantage over array: dynamic size, efficient insertions/deletions at known position.
  • Disadvantage: no random access, extra memory for pointers, poor cache performance.

Stack & Queue

Stack infix/postfix conversion and queue types are very frequently tested.

Stack — LIFO (Last In, First Out)

  • Operations: Push, Pop, Peek/Top, isEmpty — all O(1)
  • Overflow: pushing to full stack
  • Underflow: popping from empty stack

Stack Applications

  • Infix → Postfix / Prefix conversion
  • Postfix expression evaluation
  • Balanced parenthesis checking Hot
  • Function call management (activation records)
  • DFS graph traversal (iterative)
  • Undo / Redo operations
  • Tower of Hanoi
  • Backtracking algorithms

Queue — FIFO (First In, First Out)

  • Operations: Enqueue, Dequeue, Front, Rear — all O(1)
  • Front pointer: dequeue side
  • Rear pointer: enqueue side

Types of Queue

Circular Queue

Solves wasted space in simple queue. Rear wraps around to front. Condition: (rear+1)%n == front.

Priority Queue

Elements dequeued by priority, not order. Implemented using heap. Used in Dijkstra's, Prim's algorithms.

Deque (Double-Ended)

Insert/delete from both ends. Input-restricted or output-restricted deque.

Applications

BFS traversal, CPU scheduling (Round Robin), I/O buffers, Printer spooling.

Infix → Postfix Conversion Rules Very Frequently Asked

Operator Precedence (High → Low): ^ (right-assoc) > * / > + - > (

Algorithm Steps

  1. Scan expression left to right.
  2. Operand → immediately output it.
  3. Open bracket ( → push on stack.
  4. Operator: pop and output operators with higher or equal precedence, then push.
  5. Close bracket ) → pop until ( is found; discard both brackets.
  6. End of expression → pop and output remaining stack elements.
Worked Example

A + B * C - D → Postfix?

Scan A → output A
Scan + → push +
Scan B → output B
Scan * → * > + so push *
Scan C → output C
Scan - → pop * (output), pop + (output), push -
Scan D → output D
End → pop - (output)
Result: A B C * + D -
Infix → Prefix

Reverse the expression, swap ( and ), convert to postfix, then reverse result.

Trees

Binary tree properties, BST, AVL rotations, Heap — all heavily tested.

Binary Tree Properties — Memorise All!

Key Formulas

Max nodes at level i  =  2ⁱ (root = level 0)
Max nodes in height h  =  2^(h+1) – 1
Min height with n nodes  =  ⌊log₂n⌋
Max height with n nodes  =  n – 1 (skewed tree)
Leaf nodes  =  Internal nodes + 1 (full binary tree)
Total nodes (full BT, L leaves)  =  2L – 1

Tree Traversals

TraversalOrderUse case
InorderLeft → Root → RightBST sorted output
PreorderRoot → Left → RightTree copy, prefix
PostorderLeft → Right → RootTree deletion, postfix
Level-orderLevel by level (BFS)Shortest path in BT

Given any two traversals (inorder + preorder or inorder + postorder), tree can be uniquely reconstructed.

Types of Binary Trees

Full (Strictly) BT

Every node has exactly 0 or 2 children. No node has exactly 1 child.

Complete BT

All levels filled except possibly the last. Last level filled from left to right. Used for Heap implementation.

Perfect BT

All internal nodes have 2 children AND all leaves are at the same level. A perfect BT with height h has 2^(h+1)–1 nodes.

Balanced BT

Height of left and right subtrees differ by at most 1 for every node. Height = O(log n).

Degenerate (Skewed)

Every internal node has one child. Essentially a linked list. Height = n–1. BST operations become O(n).

Extended BT

Every missing child replaced by a special external (null) node. Used in analysing 2-3 trees.

Binary Search Tree (BST)

BST Property

  • Left subtree keys < Root key
  • Right subtree keys > Root key
  • Inorder traversal = sorted order
  • No duplicate keys (typically)
Search / Insert / Delete  =  O(h)
Average (random): O(log n)
Worst (skewed): O(n)

BST Deletion — 3 Cases

  1. Node is a leaf → simply remove it.
  2. Node has one child → replace node with its child.
  3. Node has two children → replace with inorder successor (smallest in right subtree) or inorder predecessor (largest in left subtree), then delete that node.

AVL Tree (Height-Balanced BST) Very Hot Topic

Balance Factor (BF) = Height(left subtree) – Height(right subtree). BF must be –1, 0, or +1 for every node.

Height of AVL tree with n nodes is always O(log n). Guaranteed O(log n) for all operations.

Heap

Heap Properties

  • Max-Heap: parent ≥ children (root = max)
  • Min-Heap: parent ≤ children (root = min)
  • Always a complete binary tree
  • For array representation: parent of i = ⌊(i-1)/2⌋, left child = 2i+1, right child = 2i+2

Heap Complexities

OperationTime
Insert (heapify up)O(log n)
Extract max/min (heapify down)O(log n)

Heap sort (n elements)O(n log n)
Peek (max or min)O(1)

Build heap = O(n), NOT O(n log n) — common trick question!

🔥 UGC NET Hot Points — Trees
  • Build heap is O(n) — not O(n log n). Frequently asked as MCQ trap.
  • Minimum number of nodes in AVL tree of height h: N(h) = N(h-1) + N(h-2) + 1, where N(0)=1, N(1)=2.
  • B-Tree: All leaves at same level. Minimum degree t → each node has 2t–1 max keys, t–1 min keys (except root).
  • Threaded binary tree: NULL pointers replaced by threads to inorder predecessor/successor for faster traversal.
  • In a complete binary tree with n nodes: number of leaf nodes = ⌈n/2⌉.

Graphs

BFS/DFS, Dijkstra's, Floyd-Warshall, Kruskal's, Prim's — all come in UGC NET.

Graph Representations

Adjacency Matrix

  • V×V matrix; cell [i][j]=1 if edge i→j exists
  • Space: O(V²)
  • Edge check: O(1)
  • Adding vertex: O(V²) (resize matrix)
  • Best for: dense graphs

Adjacency List

  • Array of V lists; each list holds neighbours
  • Space: O(V + E)
  • Edge check: O(degree)
  • Adding vertex: O(1)
  • Best for: sparse graphs (real-world networks)

Graph Traversals

BFS (Breadth-First Search)

  • Uses a Queue
  • Explores level by level
  • Time: O(V + E)   Space: O(V)
  • Finds shortest path in unweighted graphs
  • Produces a BFS tree (spanning tree)
  • Applications: shortest path, social networks, web crawling, peer-to-peer networks

DFS (Depth-First Search)

  • Uses a Stack (or recursion)
  • Explores as deep as possible first
  • Time: O(V + E)   Space: O(V)
  • Applications: topological sorting, cycle detection, SCC (Kosaraju's/Tarjan's), maze solving, backtracking

Shortest Path Algorithms

AlgorithmTime (basic)Time (with heap)Negative weights?Type

Floyd-Warshall O(V³) — Yes All pairs shortest path

Minimum Spanning Tree (MST)

Kruskal's Algorithm

  • Sort all edges by weight: O(E log E)
  • Add edge if it doesn't form a cycle (use Union-Find)
  • Total: O(E log E) or O(E log V)
  • Best for: sparse graphs
  • Uses: greedy approach on edges

Prim's Algorithm

  • Start from any vertex, grow MST greedily
  • With adjacency matrix: O(V²)
  • With priority queue + adj list: O(E log V)
  • Best for: dense graphs
  • Uses: greedy approach on vertices
🔥 UGC NET Hot Points — Graphs
  • A graph with V vertices and V-1 edges that is connected = a Tree.
  • Topological sort is only possible for DAG (Directed Acyclic Graph). Uses DFS.
  • Maximum edges in undirected graph: V(V-1)/2. In directed: V(V-1).
  • Euler circuit exists if and only if all vertices have even degree.
  • Euler path exists if exactly 0 or 2 vertices have odd degree.
  • Dijkstra's fails with negative weights — use Bellman-Ford instead.
  • A complete graph Kn has n(n-1)/2 edges.

Critical Insights

  • Best general-purpose: Quick Sort (avg O(n log n), in-place)
  • Guaranteed O(n log n): Merge Sort & Heap Sort
  • External sorting: Merge Sort (disk I/O friendly)
  • Nearly sorted data: Insertion Sort is best (O(n))
  • Only O(1) space + O(n log n) all cases: Heap Sort
  • Linear time (when possible): Counting, Radix, Bucket
  • Quick Sort worst case: already sorted array + first/last pivot

Stability — What Is It?

A sort is stable if equal elements maintain their original relative order after sorting.

Stable: Bubble, Insertion, Merge, Counting, Radix, Bucket
Not Stable: Selection, Quick, Heap, Shell

Trick: "BIM CRB" = stable sorts. Stability matters when secondary keys must preserve order.

Comparison-based Lower Bound

Min comparisons needed to sort n elements = Ω(n log n)
Proof: decision tree has n! leaves, height ≥ log₂(n!)
🔥 Tricky UGC NET Questions on Sorting
  • Quick Sort: worst case O(n²) but average O(n log n) with random pivot — median-of-3 prevents worst case.
  • Selection Sort makes exactly n-1 swaps (regardless of data) — minimum swaps property.
  • Bubble Sort in best case with optimization flag: O(n) — only 0 swaps needed for sorted array.
  • Merge Sort: divide step = O(1), merge step = O(n), depth = O(log n) → total O(n log n).
  • Which sorting algorithms are NOT comparison-based? → Counting, Radix, Bucket sort.

Searching & Hashing

Binary search conditions, hash functions, and collision resolution — all come in UGC NET.

Linear Search

  • Works on: unsorted AND sorted arrays
  • Best: O(1) (element at first position)
  • Average/Worst: O(n)
  • No pre-processing needed

Binary Search

  • Requires: sorted array
  • Best: O(1)
  • Average/Worst: O(log n)
  • Divide search space in half each step
  • Recurrence: T(n) = T(n/2) + O(1)
mid = low + (high – low) / 2
(avoids integer overflow vs (low+high)/2)

Hashing

Hash Functions

Division Method: h(k) = k mod m    (m = table size, m = prime)
Multiplication Method: h(k) = ⌊m × (k·A mod 1)⌋   A ≈ 0.618
Mid-Square: Square the key, take middle digits as index

Load factor α = n/m (n = items, m = slots). Lower α = fewer collisions.

Hashing Complexities

OperationAverageWorst
SearchO(1)O(n)
InsertO(1)O(n)
DeleteO(1)O(n)

Worst case O(n) when all keys hash to the same slot (full collision).

Collision Resolution Techniques

Chaining (Open Hashing)

  • Each slot holds a linked list
  • Multiple keys can share same slot
  • No overflow if table has capacity in lists
  • Average search: O(1 + α)
  • Load factor can exceed 1
  • Simple to implement, but extra memory

Open Addressing (Closed Hashing)

All keys stored in the table itself. Load factor must stay ≤ 1.

Linear Probing: h(k,i) = (h(k) + i) mod m
— Easy but causes primary clustering

Quadratic Probing: h(k,i) = (h(k) + c₁i + c₂i²) mod m
— Reduces clustering; secondary clustering exists

Double Hashing: h(k,i) = (h₁(k) + i·h₂(k)) mod m
— Best method, almost no clustering
Key Hashing Facts for Exam
  • Best m (table size): a prime number far from powers of 2 (for division method).
  • Clustering: Linear probing → primary clustering. Quadratic → secondary clustering. Double hashing → no clustering.
  • Perfect hashing: O(1) worst case — requires advance knowledge of all keys.
  • Load factor α > 0.7 significantly increases collision chance in open addressing. Rehash at α ≈ 0.7.

Dynamic Programming vs Greedy vs Divide & Conquer

Dynamic Programming

  • Optimal substructure + overlapping subproblems
  • Memoization (top-down) or Tabulation (bottom-up)
  • 0/1 Knapsack: O(nW)
  • LCS: O(mn), LIS: O(n²)
  • Matrix Chain: O(n³)
  • Floyd-Warshall: O(V³)

Greedy Algorithms

  • Locally optimal choice at each step
  • Greedy-choice property + optimal substructure
  • Activity Selection: O(n log n)
  • Fractional Knapsack: O(n log n)
  • Huffman Coding: O(n log n)
  • Dijkstra's, Prim's, Kruskal's

Divide & Conquer

  • Divide → Solve → Combine
  • Merge Sort: O(n log n)
  • Quick Sort: O(n log n) avg
  • Binary Search: O(log n)
  • Strassen's Matrix: O(n^2.81)
  • Recurrences solved by Master Theorem
🔥 DP vs Greedy — Favourite MCQ Trap
  • 0/1 Knapsack → Dynamic Programming (greedy fails).
  • Fractional Knapsack → Greedy works perfectly.
  • Activity Selection → Greedy (select earliest finishing time).
  • Huffman encoding produces optimal prefix-free code — proved by greedy exchange argument.
